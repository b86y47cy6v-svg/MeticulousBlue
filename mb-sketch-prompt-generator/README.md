# mb-sketch-prompt-generator

将一段文字转化为**手绘风格知识图片**的图像生成提示词——输入文本，输出 1 张结构总览图 + N 张知识点微骨架图的完整英文提示词。

## 这个 Skill 做什么

你给它一段文字（文章、口播稿、笔记、教程……），它：

1. **提炼全篇结构** → 生成 1 张结构总览图的提示词（展示整篇的逻辑骨架和脉络）
2. **拆解每个知识点** → 每个知识点生成 1 张观点图的提示词（拆解 2-4 个子要素及其关系）
3. **全自动视觉决策** — 根据内容自动选择纸张、配色、构图、图标，你不需要懂设计

最终输出可以直接粘贴到 **Midjourney / Stable Diffusion / DALL-E / 通义万相** 使用。

## 产出示例

```
输入：一篇 2000 字的「如何用 AI 搭建个人知识库」文章

输出：
  🗺 1 段结构总览图提示词 — 「知识库搭建五步法」流程图
  🖼 观点图提示词 × 4
     · 知识过滤器 — 三层漏斗微骨架
     · 下载 vs 安装 — 左右对比微骨架
     · AI 增强回路 — 循环流程图微骨架
     · 系统是长出来的 — 种子→树层级微骨架
```

## 如何使用

### 在 Claude Code 中使用

将 `SKILL.md` 放到 `~/.claude/skills/mb-sketch-prompt-generator/` 目录下，然后在对话中：

```
/sketch  [粘贴你的文字]
```

或直接说「帮我把这段文字生成视觉笔记」「生成插图」即可触发。

### 在其他 Claude 客户端中使用

将整个 `SKILL.md` 的内容复制到对话开头，然后提供你要转化的文字。

### 拿到提示词后

1. 将每段提示词分别粘贴到 Midjourney / Stable Diffusion / DALL-E / 通义万相
2. 每段提示词生成一张图
3. 总览图 + 观点图合在一起就是一套完整的知识图解

## 边界和限制

### ✅ 适合的输入

- 有明确逻辑结构的文章（观点+论据、步骤、对比、因果）
- 有多个可拆解知识点的内容
- 包含具体概念和视觉锚点的文字
- 长度：50-5000 字最佳

### ❌ 不适合的输入

- 纯粹的情绪表达或抒情散文（缺少可拆解的结构）
- 只有一个孤立的句子或金句（不足以拆解为微骨架）
- 需要精确数据可视化的图表（这是手绘风格笔记，不是信息图）

### ⚠️ 技术限制

- **只生成提示词，不生成图片**。你需要自己把提示词粘贴到图像生成工具中
- **中文渲染不完全可靠**。当前所有 AI 图像模型对中文字渲染不稳定。提示词已包含中文引导策略，但不保证每个字都正确。如乱码，可用 PS / Canva / 黄油相机后期贴中文
- **输出格式**：提示词为英文（因为主流图像模型对英文提示词响应最好），但画面的核心视觉元素是中文
- **单概念聚焦**：每张图聚焦一个知识点，不适合把多个不相关的概念硬塞进一张图

## 输出结构

```
🗺 结构总览图 ×1    → 全篇逻辑骨架（流程/层级/模块/闭环/路径）
🖼 结构观点图 ×N    → 每个知识点的微骨架（2-4 子要素 + 关系）
```

每张图都包含：手绘中文大标题 + 节点标签 + 手绘图标 + 箭头/连接线 + 牛皮纸纹理

## 视觉风格

手绘 Sketchnote 风格 — 牛皮纸或做旧白纸背景、黑色墨水线条、马克笔强调色、咖啡渍折纸边。中文是画面主角，图标是辅助。4:3 横版。
